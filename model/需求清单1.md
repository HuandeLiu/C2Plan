# 将用户输入转化为模型输入

现在的eval都是程序写死的测试用例，现在我想实现用户能够对平面图生成的条件做自定义。现在我把用户需求转化为demo.json的格式存储，接着我需要把该文件的存储内容转化为模型可输入的数据集格式，参考rplanhg_[datasets.py，我需要把demo.json文件做以下映射](http://datasets.py)

- name：户型图id，可以在真实的数据集processed_rplan/rplan_{set_name}_{target_set}.npz中找到对应的cond信息，并把以下信息拿到

```python
cond = {
                'door_mask': self.door_masks[idx],
                'self_mask': self.self_masks[idx],
                'gen_mask': self.gen_masks[idx],
                'room_types': self.houses[idx][:, self.num_coords:self.num_coords+25],
                'corner_indices': self.houses[idx][:, self.num_coords+25:self.num_coords+57],
                'room_indices': self.houses[idx][:, self.num_coords+57:self.num_coords+89],
                'src_key_padding_mask': 1-self.houses[idx][:, self.num_coords+89],
                'connections': self.houses[idx][:, self.num_coords+90:self.num_coords+92],
                'room_areas':self.houses[idx][:, self.num_coords+92:self.num_coords+92+1],
                'graph': graph,
                'boundary': boundary,
                # 'distance': boundary[3:]
                }
```

- room_type：可以映射为syn_room_types
- room_corner_nums：每个房间的拐点数量，可以推导出 syn_self_mask、syn_gen_mask、syn_src_key_padding_mask、syn_connections、syn_corner_indices、syn_room_indices
- room_area_rate：可以推导出 syn_room_areas
- room_connections：记录每个房间和门的连接情况，可以推导出syn_graph、syn_door_mask
