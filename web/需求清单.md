# 平面图生成系统全栈开发需求
## 一、系统整体架构
本系统为前后端分离架构，核心流程：
1、用户在前端上传边界图，可视化配置房间属性和连接关系
2、前端完成数据校验后，将边界图文件 + JSON 配置发送给后端
3、后端保存文件并创建异步任务，立即返回任务 ID
4、后端后台线程执行平面图生成模型脚本
5、前端通过轮询查询任务状态，实时展示进度
6、模型执行完成后，前端通过静态资源 URL 展示生成的平面图
核心约束：所有服务器本地文件操作（保存配置 / 边界图 / 模型输出）均由后端完成，前端仅通过 HTTP 接口与后端交互
## 二、前端页面需求
一、页面核心定位

该页面是平面图生成系统的用户自定义需求输入界面，核心目标是让用户可视化配置户型边界图、房间属性（类型 / 拐点数量 / 面积占比）、房间连接关系，最终将配置内容格式化为指定 JSON 结构。页面需兼顾专业性、易用性与视觉美观性，适配 PC 端主流分辨率（响应式设计）。

二、整体设计风格

- 视觉风格：现代简约 + 轻量专业，贴合室内设计工具调性；主色调选用「深蓝（#165DFF）+ 浅灰（#F5F7FA）」，辅助色「浅绿（#36D399）」（成功态）、「橙红（#FF7D00）」（提示态），文字色「深灰（#333333）/ 中灰（#666666）」，整体配色柔和不刺眼，层级清晰。
- 交互风格：以「可视化配置」为主、「文本输入」为辅，操作流程线性清晰；关键操作（如保存、添加房间）有即时反馈，复杂配置项提供引导提示，支持一键重置 / 预览。

三、页面布局（左侧配置 + 右侧预览，顶部操作 + 底部备注）

| 区域 | 占比 | 核心内容                                          |
| ---- | ---- |-----------------------------------------------|
|顶部操作区|	通栏| 页面标题、「重置」「预览 JSON」「保存 JSON」按钮                 |
|左侧配置区|	60%| 分 3 个子模块：边界图配置、房间属性配置、房间连接关系配置                |
|右侧预览区|	40%| JSON 结构实时预览（语法高亮）、房间连接关系拓扑图（可视化）、下方展示生成出来的平面图 |
|底部备注区|	通栏| 房间类型映射说明、数据校验规则提示                             |

四、各模块功能与交互细节

（一）顶部操作区
- 标题：居中显示「平面图生成系统」（18px 粗体）；
- 「重置」按钮：点击清空所有配置，弹窗确认后执行，恢复初始状态；
- 「预览 JSON」按钮：点击后在右侧预览区渲染格式化的 JSON 结构（语法高亮）；
- 「保存 JSON」按钮：校验通过后生成 UUID 文件名，将 JSON 保存至user_data目录；保存成功 / 失败均弹窗提示（含路径 / 错误原因），校验不通过时按钮置灰并提示原因。
- 「生成平面图」按钮：校验通过后发送请求给后端，进入任务等待状态 按钮状态：校验不通过时「生成平面图」按钮置灰并显示错误原因

（二）左侧配置区 - 边界图配置
- 模块标题：「边界图配置」（16px 粗体 + 浅蓝分割线）；
- 输入项：用户在本地上传边界图
- 交互：
  - 上传成功后，自动读取文件名前缀（去掉扩展名）作为 JSON 的name字段
  - 在模块内展示边界图预览（最大宽度 300px，高度自适应）
  - 支持重新上传，覆盖原有文件和name字段

（三）左侧配置区 - 房间属性配置（核心模块）
- 模块标题：「房间属性配置」（16px 粗体 + 浅蓝分割线）；
- 核心载体：可编辑表格，支持「添加房间」「删除房间」「批量计算占比」；
- 表格列配置（含强制规则）：

|列名|控件类型	|交互 / 规则细节|
| ---- | ---- |----------------------------------|
| 房间序号 |	只读文本|	自动生成（从 0 递增），作为房间唯一索引|
|房间类型|下拉选择框|	选项：1 - 客厅、2 - 厨房、3 - 卧室、4 - 卫生间、5 - 阳台、6 - 通道、7 - 餐厅、8 - 书房、10 - 储藏室；【强制】前门（15）、内门（17）由系统自动生成，无需用户选择|
|房间拐点数量|	数字输入框|	默认值 4，最小值 4（不可修改）；【强制】前门 / 内门的拐点数量固定为 4，不可编辑|
|房间面积占比	|数字输入框	|输入范围 0-1（保留 2 位小数）；【强制】前门 / 内门占比固定为 0.02（2%），不可编辑|
|操作|	按钮组	|「删除」（最后一行不可删）、「设为前门」（仅可设置 1 个前门）|

（四）左侧配置区 - 房间连接关系配置
- 模块标题：「房间连接关系配置」（16px 粗体 + 浅蓝分割线）；
- 核心逻辑：A、B 房间通过 C 门连接 → 连接关系格式为[[A,C],[B,C]]（C 为内门序号），每创建一个连接关系，就相当于多了一个房间(门)；
- 交互流程：
  - 步骤 1：下拉选择「房间 A」「房间 B」（仅显示非门房间，不可重复）；
  - 步骤 2：点击「创建内门并绑定连接」，系统自动生成内门（类型 17、拐点 4、占比 0.02），并生成连接关系；
  - 已配置列表：以卡片展示「房间 A - 内门 X - 房间 B」，支持「删除连接」（同步移除内门及占比）；
  - 反馈：创建 / 删除连接后，右侧拓扑图、JSON 预览区实时更新。

（五）右侧预览区
- JSON 预览模块：
  - 标题：「JSON 结构预览」；
  - 展示：带滚动条的文本区域，JSON 语法高亮（关键字 / 数值 / 字符串区分颜色），实时同步配置修改；
- 拓扑图预览模块：
  - 标题：「房间连接关系预览」；
  - 展示：简易拓扑图（房间 = 圆形框、门 = 线条展示连接关系）
  - 交互：鼠标悬浮节点 / 线条，显示属性弹窗（如面积占比、拐点数量）。
- 模型输出平面图
  - 标题：「生成结果」
  - 状态展示：
    - 未生成：显示提示文字「点击左侧「生成平面图」按钮开始生成」
    - 生成中：显示加载动画 + 进度提示（如「模型推理中，预计还需 8 分钟...」）+「取消生成」按钮
    - 成功：展示生成的平面图（最大宽度 100%，高度自适应）+「下载图片」按钮
    - 失败：显示红色错误信息 +「重试」按钮
（六）底部备注区
- 固定展示以下说明（12px 浅灰色）：
- 房间类型映射：1 - 客厅、2 - 厨房、3 - 卧室、4 - 卫生间、5 - 阳台、6 - 通道、7 - 餐厅、8 - 书房、10 - 储藏室、15 - 前门、17 - 内门；
- 校验规则：①面积占比总和 = 1；②拐点数量≥4；③前门仅 1 个；④内门随连接关系自动生成。

五、样式与交互优化
- 控件样式：输入框 / 下拉框聚焦时边框高亮（#165DFF），错误状态边框红（#F53F3F）+ 下方提示；
- 按钮样式：默认浅蓝底白字，hover 加深底色，禁用态浅灰，点击有按压动效；
- 表格样式：奇偶行背景色区分（#F5F7FA/#FFFFFF），hover 行淡蓝底色；
- 响应式：屏幕 < 1200px 时，左右布局改为上下布局（预览区在配置区下方）；

## 三、后端接口需求
一、项目背景

该后端接口是平面图生成系统的核心枢纽，需承接前端自定义配置的 JSON 数据，完成「数据持久化→模型调用→结果返回」的全流程闭环，最终将生成的平面图图片反馈给前端展示。

二、技术栈要求
- Web 框架：FastAPI（必须，自动生成 Swagger 文档）
- 异步任务：使用 FastAPI 内置BackgroundTasks（单用户场景足够，无需 Celery）
- 工具库：uuid（生成唯一 ID）、pathlib（路径处理）、subprocess（执行脚本）、logging（日志记录）
- 必须配置 CORS 中间件，允许所有来源的请求（开发环境）

三、核心功能需求
1、文件管理
- 自动创建../user_data和../output_user目录（不存在时）
- 每个任务使用同一个 UUID v4 作为唯一标识
- 配置文件保存为../user_data/{task_id}.json
- 边界图保存为../user_data/{task_id}.png
- 模型输出目录为../output_user/{task_id}/
2、异步任务执行
- 接收请求后立即返回任务 ID，后台线程执行模型
- 模型生成平面图脚本调用命令参考，且可以修改这个文件让它通用化： ../script_user.sh 
- 脚本超时时间：10 分钟，超时则终止进程并标记任务失败
- 记录脚本的标准输出和错误日志到../output_user/{task_id}/log.txt
3、静态资源服务
- 映射../user_data到/api/user_data/{filename}
- 映射../output_user到/api/output/{task_id}/{filename}
- 支持 GET 请求访问图片和日志文件

四、接口规范
所有接口统一返回 JSON 格式，标准响应体：
```json
{
    "code": 200,       // 状态码：200成功，400客户端错误，500服务器错误
    "data": {},        // 成功时返回数据，失败时为null
    "message": "success",  // 提示信息
    "detail": ""       // 详细错误信息（失败时）
}
```
1. 生成平面图接口
- 路径：/api/generate-floor-plan
- 方法：POST
- Content-Type：multipart/form-data
- 请求参数：
  - config：字符串类型，JSON 格式的户型配置
  - boundary_image：文件类型，边界图图片
- 成功响应：
```json
{
    "code": 200,
    "data": {"task_id": "xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx"},
    "message": "任务创建成功"
}
``` 
2. 查询任务状态接口
- 路径：/api/task/{task_id}
- 方法：GET
- 任务状态定义：
  - pending：排队中
  - running：执行中
  - success：成功
  - failed：失败
- 成功响应（任务完成）：
```json
  {
      "code": 200,
      "data": {
          "task_id": "xxx",
          "status": "success",
          "image_url": "/api/output/xxx/pred_b/0c_pred.png",
          "log_url": "/api/output/xxx/log.txt"
      },
      "message": "生成成功"
  }
```
3. 取消任务接口
- 路径：/api/task/{task_id}/cancel
- 方法：POST
- 功能：终止正在执行的脚本进程，标记任务为取消状态

（四）错误处理
必须覆盖以下异常场景，并返回对应错误信息：
- 400：请求参数缺失、JSON 格式错误、业务规则校验失败
- 404：任务不存在
- 500：文件保存失败、脚本不存在、脚本执行失败、超时、图片未生成

（五）日志规范
- 所有日志必须包含task_id，便于追踪单个请求
- 记录关键步骤：接收请求、文件保存成功、脚本开始执行、脚本执行完成、任务结束
- 错误日志必须包含完整的异常堆栈信息

五、技术栈强制要求
- 前端：Vue3 + Vite + Element Plus（使用 Element Plus 的所有基础组件：表格、按钮、弹窗、上传、进度条等）
- 后端：Python 3.9+ + FastAPI 0.100+ + Uvicorn
- 代码要求：
  - 结构清晰，模块化组织
  - 关键逻辑添加详细注释
  - 变量和函数命名规范（驼峰式前端，蛇形后端）
  - 无硬编码路径和魔法数字
  - 可直接运行，无需额外修改核心逻辑

六、交付物要求
- 前端完整项目：
  - 包含package.json（所有依赖版本明确）
  - 包含vite.config.js配置
  - 单文件组件结构，按功能拆分模块
  - 启动命令：npm run dev
- 后端完整项目：
  - 包含requirements.txt（所有依赖版本明确）
  - 单文件main.py（逻辑清晰，不拆分多个文件）
  - 启动命令：uvicorn main:app --reload --host 0.0.0.0 --port 8000
  - 自动生成 Swagger 文档：http://localhost:8000/docs
- 部署说明文档：
  - 前后端环境搭建步骤
  - 启动命令
  - 接口测试方法
  - 常见问题排查（如脚本权限、路径问题）
